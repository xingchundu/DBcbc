# DBcbc 二进制安装包说明（dbcbc-2.0.0-bin.zip）

本文档说明如何使用 **`dbcbc-2.0.0-bin.zip`** 在本地安装并启动 DBcbc（数据库同步服务）。

---

## 一、环境要求

| 项目 | 说明 |
|------|------|
| **操作系统** | Linux x86_64 / aarch64、macOS（与启动脚本支持一致）；Windows 可使用 `bin` 目录下的对应脚本 |
| **JDK** | **JDK 1.8**（路径需加入环境变量，或使用脚本内注释方式指定 `JAVA_HOME`） |
| **内存** | 默认 JVM 约 **3GB** 堆（`-Xms3g` / `-Xmx3g`），机器物理内存建议不少于 **4GB** |
| **磁盘** | 预留日志与数据目录空间（解压目录下 `logs`、`data` 等会随运行增长） |

---

## 二、解压安装

1. 将 **`dbcbc-2.0.0-bin.zip`** 复制到目标服务器（或本机）的安装目录，例如：
   - Linux：`/opt/dbcbc`
   - Windows：`D:\Programs\dbcbc`
2. **解压** ZIP，得到一个根目录（通常名为 `dbcbc-2.0.0`，以实际解压结果为准）。
3. 确认目录结构大致包含：
   - `bin/`：启动、停止脚本（如 `startup.sh`、`startup.bat`、`stop.sh` 等）
   - `conf/`：配置文件（如 `application.properties`）
   - `lib/`：程序与依赖 Jar

> **说明**：若解压后根目录名称与上文不同，以下路径中的 **`安装根目录`** 均指解压后的顶层目录。

---

## 三、配置（可选）

解压后若已有 **`conf/application.properties`**，可按需修改，常见项包括：

- **`server.port`**：Web 管理服务端口（默认 **`20268`**）。
- **`server.ip`**（若配置）：监听地址。
- 其他数据源、插件等按需阅读文件内注释调整。

修改前建议备份 **`conf/application.properties`**。

Linux / macOS 若未全局配置 JDK，可在 **`bin/startup.sh`** 中取消注释并设置：

```bash
#JAVA_HOME=/opt/jdk1.8.0_202
```

指向本机 JDK 8 的安装路径。

---

## 四、启动服务

### Linux / macOS

```bash
cd 安装根目录/bin
chmod +x startup.sh stop.sh    # 若无可执行权限时执行一次
./startup.sh
```

看到类似 **`Start successfully! PID: xxxx`** 表示启动脚本已拉起进程。

### Windows

解压后进入 **`安装根目录\bin`**，双击或命令行执行：

```cmd
startup.bat
```

或使用同目录 **`startup.cmd`**（若存在）。

### 校验运行

浏览器访问：**http://127.0.0.1:20268**（若改了端口则用新端口）。

- 默认账号：**admin**
- 默认密码：**admin**

首次登录后建议立即修改密码。

---

## 五、停止服务

Linux 可使用（以实际脚本为准）：

```bash
cd 安装根目录/bin
./stop.sh
```

也可根据进程主类 **`org.dbcbc.web.Application`** 按运维规范结束进程。Windows 请使用包内提供的停止方式或任务管理器结束对应 Java 进程。

---

## 六、目录与日志

| 路径（相对安装根目录） | 用途 |
|------------------------|------|
| `logs/` | 运行日志、错误日志等 |
| `conf/` | 配置文件 |
| `lib/` | 程序库 |
| `tmp.pid` | 启动脚本可能写入的进程 PID（以实际为准） |

启动失败时，请先查看 **`logs`** 下最新日志；OOM 时可能生成 **`logs/heapdump.hprof`**（与默认 JVM 参数相关）。

---

## 七、端口与防火墙

- 默认对外提供 Web 服务的端口为 **`20268`**。
- 若需远程访问，请在操作系统防火墙或云安全组中 **放行对应 TCP 端口**。

---

## 八、常见问题

1. **提示找不到 `JAVA_HOME` 或 `java`**  
   安装 JDK 8 并配置 `PATH` / `JAVA_HOME`，或在 **`startup.sh`** 中指定 `JAVA_HOME`。

2. **端口被占用**  
   修改 **`conf/application.properties`** 中的 **`server.port`**，或关闭占用端口的进程后重启。

3. **内存不足**  
   在 **`bin/startup.sh`** 中适当调小 `-Xms` / `-Xmx`（需与机器内存匹配，过小可能影响大表同步）。

4. **MySQL 5.x 连接问题**  
   参考主项目 README：可能需要替换兼容的 MySQL 驱动 Jar 到 `lib` 目录。

## 九、Oracle 源库增量同步注意事项（如 Oracle → PostgreSQL）

以下面向 **Oracle 作为源、开启增量** 的场景。**PostgreSQL 仅作为增量应用的目标**，通过 JDBC 写入；Oracle 一端通过 **LogMiner（`DBMS_LOGMNR`）** 从 redo / 归档解析变更，**断点续传位点为 SCN**。全量仍以 Oracle 连接器正常读表为准。

### 9.1 实例与库级配置

| 项目 | 说明 |
|------|------|
| **归档模式** | 依赖重做/归档的持续增量一般需要 **`ARCHIVELOG`**（`SELECT LOG_MODE FROM V$DATABASE` 可查）。`NOARCHIVELOG` 下通常无法满足持续增量。 |
| **补充日志** | LogMiner 重放 **`UPDATE`/`DELETE`** 等往往需要 **合适的 supplemental logging**（库级或与表相关的 **主键补充日志**，按实际表结构设计）。缺失时易出现无法唯一定位行、解析不完整等问题。 |
| **网络与连接** | 放行监听端口、`SERVICE_NAME`/SID 与 JDBC 一致；**RAC/PDB/CDB** 时连接串、会话容器需与实际部署一致（增量逻辑可能涉及 CDB 根容器会话，请以环境实测为准）。 |
| **与目标库差异** | 字符集、时区、`TIMESTAMP WITH TIME ZONE`、大字段与类型映射等可能影响落地，详见 **`README.md`** 中 Oracle 与目标库（如 PostgreSQL）相关说明。 |

### 9.2 同步用 Oracle 账号权限（程序启动时会校验）

满足 **其一** 即可作为“角色”侧快捷通过：**拥有 `DBA` 角色**则不再逐项检查；否则 **必须** 授予 **`SELECT_CATALOG_ROLE`**。

此外，当前会话需具备以下 **系统权限**（在 `SESSION_PRIVS` 中均可看到）：

| Oracle 大版本 | 所需权限 |
|---------------|----------|
| **11g 及以下** | `CREATE SESSION`、`SELECT ANY TRANSACTION`、`SELECT ANY DICTIONARY` |
| **12c 及以上** | 上述三项 + **`LOGMINING`** |

由管理员执行时可参考（将 `your_sync_user` 换成实际用户；**11g** 时不要授权 `LOGMINING`）：

```sql
GRANT SELECT_CATALOG_ROLE TO your_sync_user;
GRANT CREATE SESSION TO your_sync_user;
GRANT SELECT ANY TRANSACTION TO your_sync_user;
GRANT SELECT ANY DICTIONARY TO your_sync_user;
GRANT LOGMINING TO your_sync_user;   -- 仅 12c 及以上需要
```

若同步任务还需做 **全量、对账** 等读表操作，仍需对涉及的 **业务表/schema** 授予相应 **`SELECT`**（或通过角色集中授权），按安全规范由 DBA 收紧范围。
